Renovision

AI-Powered Property Renovation Visualisation

Renovision is an AI-powered property renovation visualisation tool for Rightmove listings. Users paste a Rightmove URL, select a room, configure a renovation style, and watch AI transform the space into a realistic vision of what it could be.

Built for people who can see past magnolia walls.

The main interface - paste a Rightmove URL and select your style
Browse and select rooms from any Rightmove listing

The Problem

I built Renovision while Izzy and I were looking for a doer-upper. It solved a real problem: visualising potential without destroying the integrity of the space. I've built similar tools before, but the emergence of Nano Banana made this worth properly productionising.

The Development Journey

Product First

The philosophy was simple: ship something useful, then iterate. Every decision prioritised user value over technical elegance. The result is a tool that solves a real problem people actually have, built in a way that can evolve.

From Clerk to Postgres

Originally built with Clerk authentication for speed-to-market, I migrated to a standard PostgreSQL setup for better control, simpler data management, and lower operational costs. Sometimes the boring choice is the right choice.

Data That's Easy to Use

The database captures everything needed without overcomplicating: user sessions, property URLs, transformation configurations, and generated images. Clean foreign keys, sensible indexes, and straightforward queries. Data retrieval is fast because the schema was designed for the actual access patterns, not theoretical best practices.

Challenges Along the Way

Connectivity Issues

Rightmove's dynamic content required Playwright for reliable scraping. Initial approaches with simple HTTP requests failed on JavaScript-rendered content. The headless browser solution adds latency but ensures reliability.

Image Fidelity

Early AI outputs changed room dimensions and moved windows. The breakthrough came from strict prompt engineering: explicit rules that the AI must follow, preserving structure while transforming aesthetics.

The Breakthrough: Absolute Structural Fidelity

The core challenge — and the breakthrough — was absolute structural fidelity.

The room's dimensions, geometry, camera angle, doors, windows, and lighting must remain identical to the original photograph. The AI is only allowed to change finishes, furniture, layouts, and decor — never the bones.

This required prompt engineering that enforced discipline at every level:

  • RULE #1 – Exact Room Dimensions: The room must remain the exact same size and shape. No enlarging, shrinking, expanding, or resizing.
  • RULE #2 – Doors and Windows: Any doors or windows must remain exactly where they are — same position, size, shape, and number.
  • RULE #3 – Camera Perspective: Keep the exact same camera angle and viewpoint. Edit the photograph only — do not generate a new image.

The trick was preserving structure and lighting while allowing radical transformation through materials, furniture, and layout — making the result feel plausible, aspirational, and grounded in reality.

Privacy First
